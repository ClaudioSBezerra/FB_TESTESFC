# Milestone v1.0 Audit — FB_TESTESFC

**Audited:** 2026-07-02
**Status:** ⚠ gaps_found

## Summary

As 3 fases do milestone (Foundation, Import Pipeline & Fiscal Execution, Visual Comparison Screen) foram individualmente verificadas e aprovadas — build/vet/tsc limpos, checkpoints humanos aprovados contra Oracle real, todos os 24 requisitos (v1.0) implementados e com evidência de funcionamento via API/URL direta.

O integration-checker, ao traçar o fluxo ponta a ponta como um usuário real faria (clicando, não digitando URLs), encontrou que **a navegação da aplicação está quebrada**: não existe nenhum link clicável para nenhuma das telas de negócio construídas nas Fases 2 e 3, e usuários não-admin ficam presos num loop de redirect logo após o login. Isso nunca apareceu nos checkpoints anteriores porque toda verificação (minha e do Playwright) navegou direto pela URL, nunca através da navegação real do app.

**Isso é um bloqueador de produto, não um requisito individual não atendido** — cada peça funciona isoladamente, mas a aplicação como um todo não é usável clicando nela.

## Cross-Phase Integration Issues

### 🔴 BLOCKER-1 — Nenhuma navegação clicável para as telas de negócio

- `frontend/src/lib/navigation.ts` define `modules.config.tabs` com as 5 abas de negócio (Importar XMLs, Notas Importadas, Comparação Fiscal, + 2 de config admin).
- Esse array é importado **só** por `AppRail.tsx`, e só pra decidir qual ícone destacar — nunca é mapeado/renderizado como lista de links.
- `AppRail.tsx:34`: `const mainItems = []` — vazio desde a Fase 1 (commit `80be77c`, comentário "Fase 1: nenhum módulo de negócio"). As Fases 2 e 3 adicionaram rotas e entradas em `navigation.ts`, mas nunca adicionaram um item de navegação clicável correspondente.
- Confirmado por grep exaustivo: nenhum `<Link>`/`<a href>`/`navigate(...)` no código aponta para `/importacoes/xmls-saida`, `/importacoes/notas-saida` ou `/importacoes/comparacao-fiscal`.
- **Impacto:** todo usuário (inclusive admin) cai em `/config/erp-bridge` após login e não tem nenhum caminho descobrível pra importar um XML, rodar o cálculo fiscal ou ver a comparação — a tela que é o core value do projeto.

### 🔴 BLOCKER-2 — Loop infinito de redirect para usuários não-admin

- `Login.tsx:58` sempre chama `navigate("/config/erp-bridge")` após login, independente do `role`.
- `/config/erp-bridge` é `AdminRoute`-only (`App.tsx:63`); `AdminRoute` redireciona pra `/` se `role !== 'admin'`.
- `/` redireciona de volta pra `/config/erp-bridge` (`App.tsx:58`).
- Resultado: usuário com `role: 'user'` (criável via `AdminUsers.tsx`) entra num loop `/` ↔ `/config/erp-bridge` e nunca renderiza nada após o login.

### ⚠ Warning — Link morto

- `ERPBridgeConfig.tsx:723`: `<a href="/importacoes/erp-bridge/logs">` (navegação de página inteira, não `<Link>` do router) aponta pra uma rota inexistente, sem catch-all `*`. Baixo impacto (página admin-only), mas produz página em branco ao clicar.
