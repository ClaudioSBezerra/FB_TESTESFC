## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gaps entre ícone e texto em badges/botões pequenos |
| sm | 8px | Espaçamento compacto em linhas de tabela densa (`py-1 px-2`) |
| md | 16px | Espaçamento padrão entre campos de filtro, padding de `CardContent` |
| lg | 24px | `space-y-6` entre blocos de página (header → card de upload → card de histórico) |
| xl | 32px | Não usado nesta fase |
| 2xl | 48px | Não usado nesta fase |
| 3xl | 64px | Não usado nesta fase |

Exceptions: tabela densa usa `text-[11px]` e `py-1`/`py-1.5` (menor que a escala de 8pt em altura de linha) — **intencional e já estabelecido** no padrão herdado (`ConsultaNFeSaidas.tsx`/`ImportarXMLsSaida.tsx`) para tabelas fiscais com muitas colunas numéricas. Manter esse padrão nesta fase para as duas telas novas; não introduzir densidade diferente.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 11px (`text-[11px]`) — campos de tabela densa / 12px (`text-xs`) — labels de filtro | 500 (medium) | 1.4 |
| Heading (h1 de página) | 24px (`text-2xl`) | 700 (bold) | 1.2 |
| Heading (título de card) | 16px (`text-base`) | 600 (semibold) | 1.3 |

**Pesos declarados (máx. 2 conforme contrato):** 400 (regular, corpo/valores) e 600/700 (semibold/bold, títulos e destaques — tratados como uma única categoria "peso forte" no código herdado, que mistura `font-semibold`/`font-bold` de forma intercambiável nos títulos; não introduzir um terceiro peso nesta fase).

**Fonte:** herdado do padrão visual já em produção na Fase 1 e nas telas de referência do FB_APU04 (`ImportarXMLsSaida.tsx`, `ConsultaNFeSaidas.tsx`).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(210 20% 98%)` (`--background`) | Fundo da página, área fora dos cards |
| Secondary (30%) | `hsl(0 0% 100%)` (`--card`) | Cards de upload, histórico, filtros, tabela |
| Accent (10%) | `hsl(142 76% 36%)` (`--accent`, verde) | Reservado para: ícone/badge de sucesso (`CheckCircle`, contagem "Importados"), badge de status `ok`/item calculado sem divergência de execução |
| Destructive | `hsl(0 84% 60%)` (`--destructive`) | Reservado para: badge/ícone de erro de importação (arquivo rejeitado), badge de status `error` (falha na execução do pacote fiscal), texto de contagem "Rejeitados" |

**Cor semântica adicional usada nesta fase (já existe em `index.css`, não é nova):** `--warning` (`hsl(38 92% 50%)`, âmbar) — reservada exclusivamente para o badge de status `sem_grupo_fiscal` (ERP-03), para diferenciar visualmente "não foi possível nem tentar calcular" (âmbar) de "tentou calcular e falhou" (vermelho/destructive).

Accent reserved for:
- Ícone `CheckCircle` e badge "Importados: N" na tela de upload
- Badge de status por item = `ok` na tela de consulta
- Nunca usado em texto de corpo genérico, links ou botões primários (esses usam `--primary`, azul `217 91% 60%`, já herdado)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Não há botão CTA explícito — a dropzone de upload (`Arraste XMLs ou compactados (.zip/.rar) aqui, ou clique`) atua como o gatilho de ação, disparando o upload automaticamente ao soltar o arquivo (decisão confirmada: manter padrão herdado, sem botão "Importar" separado) |
| Empty state heading (tela de consulta) | "Nenhuma nota encontrada" |
| Empty state body (tela de consulta) | "Nenhuma nota de saída foi importada ainda. Use a tela **Importar XMLs** para carregar arquivos." |
| Empty state (histórico de upload) | "Nenhum upload registrado ainda." |
| Error state (upload — arquivo rejeitado) | "{nome do arquivo}: {motivo do erro de parse/validação}" — lista compacta, fundo `bg-red-50`, texto `text-red-700`, exibida abaixo do resultado do upload (XML-04) |
| Error state (item sem grupo fiscal) | Badge amarelo "Sem grupo fiscal" no item da tabela de consulta; tooltip/detalhe: "Produto não encontrado em PROD/PRODB — grupo fiscal não pôde ser determinado." (ERP-03) |
| Error state (falha na execução do pacote fiscal) | Badge vermelho "Erro no cálculo" no item da tabela de consulta; tooltip/detalhe: "Falha ao executar o pacote fiscal: {mensagem de erro sanitizada}." (FIS-03) |
| Toast de sucesso (upload) | "Upload concluído: {N} NF-e(s) importadas, {N} rejeitadas." (padrão herdado, mantido) |
| Toast de erro (upload) | "Erro no upload: {mensagem do backend}" |
| Destructive confirmation | Não aplicável nesta fase — nenhuma ação destrutiva (exclusão de nota/lote) está no escopo da Fase 2; decisão confirmada de deixar para fase futura |

## Telas desta Fase

### 1. Importar XMLs (rota sugerida: `/importacoes/xmls-saida`, dentro do módulo "Configurações" existente)

**Reaproveitamento:** cópia seletiva de `FB_APU04/frontend/src/pages/ImportarXMLsSaida.tsx`, adaptado para o domínio desta fase (sem `competencia`, pois não é apuração — confirmar com o planner se o campo de competência é removido ou mantido oculto).

**Contrato visual:**
- Header: `<h1 className="text-2xl font-bold tracking-tight">Importar XMLs</h1>` + subtítulo `text-sm text-muted-foreground`
- Dropzone: aceita `.xml`, `.zip`, `.rar` — limite 2GB (mantido igual ao original, decisão confirmada)
- Estados: `idle` → `scanning` → `uploading` → `polling` → `done`/`error` (idênticos ao componente original)
- Feedback por arquivo: badges verde (`Importados: N`) e vermelho (`Rejeitados: N`), com lista de erros por arquivo quando houver rejeição (XML-04)
- Progress bar (`Progress` component) durante `polling`
- Histórico de uploads abaixo, em `Card` separado, tabela densa (`text-[11px]`)

### 2. Notas Importadas (rota sugerida: `/importacoes/notas-saida`, dentro do módulo "Configurações" existente)

**Reaproveitamento:** cópia seletiva de `FB_APU04/frontend/src/pages/ConsultaNFeSaidas.tsx`, com uma coluna nova de status de execução fiscal por item.

**Contrato visual:**
- Header: `<h1 className="text-2xl font-bold tracking-tight">Notas Importadas</h1>` + subtítulo
- Filtros em `Card`: filial, cliente, intervalo de datas (padrão herdado mantido)
- Cards de totalizador (grid 2x4) acima da tabela
- Tabela principal: uma linha por nota (clicável → abre `Dialog` de detalhe com itens e valores esperados do XML — XML-03)
- **Novo nesta fase — indicador de status por item (ERP-03/FIS-03):** dentro do `Dialog` de detalhe (ou em uma sub-tabela de itens), cada item exibe um badge de status com 3 estados:

| Status | Cor/Badge | Gatilho |
|--------|-----------|---------|
| `ok` | Verde discreto (`--accent` / `bg-green-50 text-green-700 border-green-200`, mesmo padrão de `AdminUsers.tsx`) ou ausência de badge (apenas ícone `CheckCircle` pequeno) | Grupo fiscal encontrado e pacote fiscal executado com sucesso |
| `sem_grupo_fiscal` | Âmbar (`bg-yellow-50 text-yellow-700 border-yellow-200`, token `--warning`) | Lookup Oracle `prod`/`PRODB` não encontrou grupo fiscal para o produto (ERP-03) |
| `error` | Vermelho (`bg-red-50 text-red-700 border-red-200`, token `--destructive`) | Falha na execução do pacote fiscal — timeout, erro de conexão Oracle, exceção PL/SQL (FIS-03) |

- Isolamento visual: um item com status `error`/`sem_grupo_fiscal` **não bloqueia a visualização dos demais itens da mesma nota** — cada linha de item é independente na tabela/lista de detalhe.
- Esta tela **não é** a comparação item a item/imposto a imposto (isso é Fase 3) — aqui o badge apenas sinaliza status de processamento, sem exibir colunas lado a lado esperado-vs-calculado.

### 3. Configuração ERP_BRIDGE (reaproveitada, sem mudanças)

Tela `ERPBridgeCredenciais.tsx` já existente da Fase 1 — usada para configurar a conexão Oracle (ERP-01). **Nenhuma alteração de design nesta fase.** Referenciada aqui apenas para completude do fluxo (o usuário passa por ela antes de importar, se ainda não configurou credenciais).
