<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser (UI) → /api/admin/* | Telas privilegiadas; AdminRoute restringe no cliente, AuthMiddleware no servidor |
| Browser → /api/erp-bridge/* | Form de credenciais Oracle e ação de teste |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Elevation of Privilege | Rotas de admin no cliente | mitigate | `AdminRoute` esconde no cliente; a autorização real é `withAuth(..., "admin")` no backend (defense in depth) |
| T-05-02 | Information Disclosure | Exibição de credenciais Oracle | mitigate | UI exibe apenas flags `*_set`/campos mascarados; senha nunca retornada pelo backend (T-04-01) |
| T-05-03 | Spoofing | Reset de senha | mitigate | Token de reset com expiração em verification_tokens; e-mail enviado via services/email.go; resposta genérica para não enumerar contas |
| T-05-04 | Information Disclosure | Mensagem de erro do teste de conexão na UI | mitigate | Exibe a mensagem retornada pelo backend (já sanitizada, sem senha) |
</threat_model>
