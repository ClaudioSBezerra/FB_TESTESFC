<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → API (/api/auth/*) | Credenciais de login e tokens cruzam aqui (entrada não confiável) |
| API → Postgres | Queries parametrizadas; dados de auth/tenancy persistidos |
| Seed SQL → tabela users | Credencial padrão de admin gravada no provisionamento |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | POST /api/auth/login | mitigate | bcrypt cost 14 em auth.go; rate limiter `LoginRL` de middleware.go aplicado ao handler |
| T-01-02 | Elevation of Privilege | JWT (alg confusion) | mitigate | `jwt/v5` com `getJWTSecret()` HS256; `ValidateJWTSecret()` na startup aborta se ausente com DATABASE_URL setado |
| T-01-03 | Information Disclosure | Cookie refresh_token | mitigate | Cookie httpOnly + SameSite=Strict (auth.go); access token nunca persistido em disco |
| T-01-04 | Tampering | Queries Postgres | mitigate | Parâmetros `$1/$2` via lib/pq em todos os handlers copiados |
| T-01-05 | Information Disclosure | Senha padrão admin (123456) | accept | Default de dev de ferramenta interna somente-leitura (D-10); seed nunca loga a senha em claro; trocável pela tela de gestão. Endurecível em release. |
| T-01-06 | Elevation of Privilege | CORS | mitigate | `SecurityMiddleware` com whitelist `GetAllowedOrigins()`; apenas localhost:3003/3004 no fallback |
| T-01-SC | Tampering | go mod download (deps Go) | mitigate | Apenas pacotes já em produção no FB_APU04 (jwt, lib/pq, crypto, godotenv); auditoria em RESEARCH.md § Auditoria de Legitimidade; go-ora não entra neste plano |
</threat_model>
