# Phase 03.1: Fechar gap: navegação quebrada - Context

**Gathered:** 2026-07-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Fechar os 2 bloqueadores críticos encontrados no audit do milestone v1.0 (`.planning/v1.0-MILESTONE-AUDIT.md`): (1) nenhuma tela de negócio é alcançável clicando em algo na UI — as rotas existem mas não há nenhum link/ícone que leve a elas; (2) usuários não-admin caem em loop infinito de redirect logo após o login. Esta fase reestrutura o AppRail em 4 módulos navegáveis e corrige a página de destino pós-login.

**Fora desta fase:** qualquer alteração no pipeline de importação/execução fiscal (Fases 1-2) ou na tela de comparação em si (Fase 3) — esta fase só liga a "fiação" de navegação que já deveria existir. Novas páginas, novos módulos além dos 4 listados, e o link morto de `/importacoes/erp-bridge/logs` (tech debt não-crítico, ver Deferred) também estão fora.

</domain>

<decisions>
## Implementation Decisions

### Estrutura de módulos no AppRail
- **D-01:** O AppRail passa a ter **4 ícones de módulo**, substituindo o atual (só o ícone de Configurações): **Importações**, **Notas Importadas**, **Comparação Fiscal**, **Configurações**. Cada um com tooltip (reaproveitar o padrão `Tooltip` já usado no ícone de Configurações atual).
- **D-02:** **Notas Importadas** e **Comparação Fiscal** navegam direto para sua única página ao clicar no ícone (`/importacoes/notas-saida` e `/importacoes/comparacao-fiscal`, respectivamente) — sem abas, sem sub-navegação.
- **D-03:** **Importações** agrupa `Importar XMLs` (`/importacoes/xmls-saida`). Módulo com uma única página por ora, mas estruturado como módulo (não link direto) para acomodar futuras páginas de importação sem precisar reestruturar de novo.
- **D-04:** **Configurações** agrupa as páginas administrativas/de config existentes: `Ambiente` (`/config/ambiente`), `Gestores` (`/config/gestores`), `Usuários` (`/config/usuarios`, admin-only), `Credenciais ERP` (`/config/erp-bridge`, admin-only), `Config ERP` (`/importacoes/erp-bridge`, admin-only) — reaproveita a lista já presente em `modules.config.tabs` (`frontend/src/lib/navigation.ts`), sem alterar os itens, só a forma como são apresentados.
- **D-05:** Módulos com múltiplas páginas (**Importações**, **Configurações**) exibem uma **barra de abas horizontal no topo do conteúdo** ao clicar no ícone do módulo — usa a estrutura de dados `ModuleTab[]` que já existe em `navigation.ts`, só falta o componente que renderiza (`ModuleTabs`, mencionado em comentário morto no `App.tsx:54` mas nunca implementado).
- **D-06:** Itens `adminOnly: true` dentro da barra de abas de Configurações ficam ocultos/desabilitados para usuários não-admin (o campo `adminOnly` já existe no tipo `ModuleTab`, só precisa ser respeitado na renderização).

### Página de destino pós-login
- **D-07:** Após o login, **todos os usuários** (admin e não-admin) são redirecionados para **`/importacoes/comparacao-fiscal`** — é o core value do projeto (ver PROJECT.md) e a rota não tem restrição de role (`ProtectedRoute`, não `AdminRoute`), então resolve o loop de redirect automaticamente sem precisar de lógica condicional por role.
- **D-08:** A rota raiz `/` (hoje `<Navigate to="/config/erp-bridge" replace />`) passa a redirecionar para `/importacoes/comparacao-fiscal` também, pela mesma razão — consistência entre o destino do login e o destino da rota raiz.

### Claude's Discretion
- Ícones exatos de cada módulo no AppRail (Lucide icons já usados no projeto: `Settings`, `KeyRound`, etc. — escolher ícones semanticamente coerentes: ex. upload/file para Importações, list/table para Notas, git-compare/scale para Comparação Fiscal).
- Estilo visual exato da barra de abas horizontal (cores, espaçamento) — seguir o design system já estabelecido nas Fases 1-3 (Tailwind + shadcn/ui, mesmos tons neutros/azuis já em uso).
- Como o AppRail indica visualmente qual módulo está ativo (reaproveitar `getActiveModule` de `navigation.ts`, hoje subutilizado — só retorna `'config'` sempre).
- `getActiveModule` precisa ser reescrito para os 4 módulos (hoje só existe o módulo `config`); a estrutura de `modules` em `navigation.ts` precisa virar `Record<string, ModuleConfig>` com 4 chaves em vez de 1.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `Tooltip`/`TooltipContent`/`TooltipTrigger`/`TooltipProvider` (já importados em `AppRail.tsx`) — reaproveitar para tooltips dos 4 novos ícones de módulo.
- `ModuleTab`/`ModuleConfig` interfaces (já existem em `navigation.ts`) — só precisam de mais entradas em `modules` e um componente que as renderize.
- Ícones Lucide já em uso no projeto (`Settings`, `KeyRound`, `LogOut`) — biblioteca já é dependência, escolher ícones adicionais da mesma lib.

### Established Patterns
- `useLocation`/`useNavigate` do `react-router-dom` já usados em `AppRail.tsx` para navegação programática e detecção de rota ativa.
- Roteamento com `ProtectedRoute`/`AdminRoute` wrappers em `App.tsx` — nenhuma mudança nesse padrão, só nos redirects que apontam para as rotas.

### Integration Points
- `AppRail.tsx` é renderizado dentro de `AppLayout` (usado por todas as rotas protegidas) — mudança nele afeta a navegação global, não uma página isolada.
- Novo componente `ModuleTabs` (a criar) precisa ser renderizado condicionalmente dentro do layout, visível só quando o módulo ativo tem mais de uma página (Importações, Configurações) e oculto para módulos de página única (Notas Importadas, Comparação Fiscal).

</code_context>
